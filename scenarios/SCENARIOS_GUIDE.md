## 1️⃣ AI SaaS 场景 - 通过案例

### 📄 文件
`ai_saas_pass.json`

### 🏢 公司信息
- **公司名称**: 智语科技
- **行业**: AI 客服 SaaS
- **产品**: 基于大模型的智能客服系统
- **营收**: ARR 500万人民币
- **团队**: 15人，核心团队来自阿里、腾讯
- **融资需求**: A轮 2000万人民币

### ✅ 为什么应该通过

#### 强大的团队背景
- CEO: 前阿里巴巴 P9，10年 AI 研发经验
- CTO: 前腾讯 T4，AI 算法专家
- COO: 前字节跳动运营总监

#### 优秀的商业数据
- **MRR**: 42万人民币
- **增长率**: 30% MoM
- **流失率**: 5%（行业优秀水平）
- **毛利率**: 85%
- **LTV/CAC**: 15倍（健康比例）

#### 头部客户验证
- 阿里巴巴
- 腾讯
- 字节跳动
- 美团
- 京东

#### 技术优势
- 基于最新 GPT-4 模型
- 准确率 95%（行业领先）
- 响应时间 2秒
- 支持多语言和多平台集成

#### 市场机会
- TAM: 1000亿人民币
- 已有明确的市场定位和竞争优势
- 增长迅速，有望快速占领市场

### 🎯 测试目标
验证 Agent 能够识别：
- 优秀的团队背景
- 健康的商业指标
- 头部客户背书
- 技术领先优势
- 清晰的增长路径

## 2️⃣ 硬件创业场景 - 拒绝案例

### 📄 文件
`hardware_reject.json`

### 🏢 公司信息
- **公司名称**: 未来机器人
- **行业**: 消费级硬件
- **产品**: 家庭服务机器人
- **营收**: 0
- **团队**: 5人，无相关行业经验
- **融资需求**: 天使轮 500万人民币

### ❌ 为什么应该拒绝

#### 团队经验不足
- 创始人: 应届毕业生，无行业经验
- CTO: 2年开发经验，无硬件背景
- 团队无硬件、供应链、制造经验

#### 无核心技术壁垒
- 硬件外购（机械臂、传感器）
- 软件基于开源 ROS 系统
- AI 调用第三方 API
- 无自主研发能力

#### 市场竞争激烈
- 竞争对手: 科沃斯、石头科技、云鲸、追觅、小米
- 市场已被头部品牌占据
- 价格战激烈
- 无差异化优势

#### 商业模式不清晰
- 无营收、无客户
- 成本结构不明确
- 盈利路径不清晰
- 资金需求大，回报周期长

#### 高风险因素
- 技术风险: 高
- 市场风险: 高
- 团队风险: 高
- 供应链风险: 高
- 执行风险: 高

### 🎯 测试目标
验证 Agent 能够识别：
- 团队经验不足的红旗
- 缺乏核心技术壁垒
- 市场竞争过于激烈
- 商业模式不成立
- 多重高风险因素

## 3️⃣ 医疗 AI 场景 - 边界案例

### 📄 文件
`medical_ai_edge.json`

### 🏢 公司信息
- **公司名称**: 医智科技
- **行业**: 医疗 AI
- **产品**: AI 辅助诊断系统（肺结节检测）
- **营收**: ARR 200万人民币
- **团队**: 8人，医疗+AI 复合背景
- **融资需求**: Pre-A 1000万人民币

### ⚠️ 为什么是边界案例

#### 优势因素 ✅

**监管资质**
- 已获得 NMPA 二类医疗器械注册证
- ISO 13485 质量管理体系认证
- 完成多中心临床验证

**专业团队**
- CEO: 协和医院影像科副主任医师
- CTO: 清华大学 AI 博士
- 医疗+AI 复合背景

**客户验证**
- 12家三甲医院（协和、301等）
- 正在洽谈 20+ 家医院
- 有头部医院背书

**技术指标**
- 准确率: 92% 敏感度，95% 特异度
- 自研深度学习模型
- 10万+ 标注数据

#### 风险因素 ⚠️

**监管风险**
- 三类证申请周期长（2-3年）
- 审批不确定性高
- 监管要求严格

**商业化挑战**
- 营收规模小（200万）
- 增长速度一般（15% MoM）
- 销售周期长（6-12个月）
- 医保收费政策不明确

**市场竞争**
- 头部公司优势明显（推想、依图等）
- 资源和品牌差距大
- 市场集中度高

**资金压力**
- 烧钱率高（80万/月）
- 跑道短（8个月）
- 需要持续投入研发

### 🤔 决策考量因素

1. **三类证进展**: 能否按计划推进？
2. **医保政策**: 收费政策何时落地？
3. **销售增长**: 能否加速客户获取？
4. **技术提升**: 准确率能否持续优化？
5. **团队执行**: 能否平衡研发和商业化？

### 🎯 测试目标
验证 Agent 能够：
- 识别复杂的优劣势因素
- 权衡多个决策维度
- 提出关键问题和考量点
- 给出有条件的建议
- 展现深度思考能力

## 🔧 自定义场景

### 基本结构

```json
{
  "scenario_name": "场景唯一标识",
  "company_name": "公司名称",
  "industry": "行业分类",
  "product": "产品描述",
  "revenue": "营收情况",
  "team": "团队信息",
  "funding_need": "融资需求",
  "expected_result": "passed/rejected/pending",
  "project_details": {
    // 详细信息（可选）
  }
}
```

### 必填字段
- `scenario_name`: 场景名称
- `company_name`: 公司名称

### 推荐字段
- `industry`: 帮助 Agent 理解行业背景
- `product`: 产品描述
- `revenue`: 营收数据
- `team`: 团队信息
- `funding_need`: 融资需求
- `expected_result`: 用于验证测试结果

### 详细信息
`project_details` 可以包含任意 JSON 结构，例如：
- 团队详情
- 财务数据
- 技术指标
- 市场分析
- 客户列表
- 里程碑
- 风险因素

## 🐛 故障排查

### 场景加载失败
- 检查 JSON 格式是否正确
- 验证必填字段是否存在
- 确认文件编码为 UTF-8

### 测试结果不符合预期
- 查看完整对话日志
- 分析 Agent 的推理过程
- 检查是否遗漏关键信息
- 考虑调整场景描述

### Agent 表现异常
- 检查 LLM 模型配置
- 验证 system prompt 设置
- 查看 API 调用日志
- 确认网络连接正常
